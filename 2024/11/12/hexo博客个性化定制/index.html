hexo博客个性化定制

十一月 12, 2024 项目学习 本文总阅读量次

基于hexo博客搭建激发的“如何迅速的上手陌生框架及项目，并进行个性化修改”的思路整理。

hexo技术栈整理

搭建一个hexo主题通常需要掌握以下技术栈和相关知识：

  1. HTML/CSS
  2. JavaScript
  3. Hexo API
    理解 Hexo 的主题框架和 API，能够使用 Hexo 提供的 API 来获取数据（如文章、分类、标签等）并进行渲染。
  4. Nunjucks 或 EJS
  5. Markdown
  6. 配置文件
  7. 调试工具

可以发现，Hexo API外，其他的都是前端工程师必备的基础技能。那么进行hexo博客项目的深度学习时，除了强化自己的基础知识，更重要的便是快速上手一个新的框架和API。

hexo项目目录结构

成型的项目一般拥有繁杂但井然有序的目录结构，学习了解每个文件的具体效用，才能有下手可改之处。
your-hexo-blog/
│
├── .deploy_git/ # 用于 Git 部署的文件夹
├── .git/ # Git 管理的目录
├── node_modules/ # Node.js 依赖的模块
├── scaffolds/ # 模板，用于生成新文章的样板
│ └── post.md # 文章模板
│
├── source/ # 源文件目录，存放所有网站内容
│ ├── _posts/ # 文章目录，存放 Markdown 格式的文章
│ │ └── my-first-post.md
│ ├── _drafts/ # 草稿目录，存放尚未发布的文章
│ ├── _config.yml # 站点配置文件
│ ├── 404.html # 自定义 404 页面
│ └── css/ # 自定义 CSS 文件
│
├── themes/ # 存放主题文件的目录
│ └── your-theme/ # 主题文件夹
│ ├── layout/ # 主题的布局文件
│ ├── script/ # 主题的脚本文件
│ ├── source/ # 主题特有的静态资源
│ └── _config.yml # 主题配置文件
│
├── _config.yml # 全局配置文件
├── package.json # 项目描述和依赖信息
├── public/ # 生成后的静态文件目录
└── README.md # 项目说明文档

由上方的目录结构可知，对于hexo项目，如需更改网页标题、作者、语言、时区等，均可在_config.yml全局配置文件中修订，且此文件可读性较高，无需过度深挖项目即可迅速完成修改。同时在修改过程中，利用hexo s预览修改效果，每次修改只需刷新页面，无需重新hexo g。

需求发现及实现

在实际主题使用过程中，也发现了诸多需求无法简单在_config.yml中完成修改。由于本人暂时没有需求全新自制hexo主题，只在hexo主题现有基础上进行修改，深挖hexo API性价比不高，所以在本项目中，追求高效迅速完成定制。

简单思路

最简单的实现思路：根据需求特性，思考对应代码段的特点代码句，使用VScode跨文件搜索功能查找字段，完成对相应功能的修改。
